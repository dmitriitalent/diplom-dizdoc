## Идея
Основная идея заключается в создании экосистемы, которая решает проблемы проживающих, работников, комендатуры и администрации. На платформе можно будет зарегистрировать общежитие. В него как админ добавляется комендатура общежития. Проживающие регистрируют свой профиль, самостоятельно заполняя его. За ними остаётся право не указывать свои личные данные (заполнение полей не обязательно). Если пользователь не хочет в системе афишировать своё ФИО, то указывает, например, придуманный ник. 

Студент будет верифицироваться по студенческой почте. Проживающий будет верифицироваться по карточке-пропуску в общежитие (фотка лица рядом с карточкой).

Далее по тексту предполагается, что интеграция с комендатурой, а проживающие деанонимизированы и все принудительно регистрируются.
1. + Чат комнаты, этажа, корпуса, здания.
2. + Лента новостей общежития.
3. + Расписание работы душа/комендатуры/смены белья.
4. + Каталог раздачи ненужных вещей
5. + Афиша предоставления платных услуг
6. + Афиша общих активностей
7. ++ Заполнение журнала использования стиральной машины.
8. +++ Личный чат комендатуры и проживающего.
9. +++ Прикрепление документов, справок
10. ++++ Автоматический доступ к документам/справкам администрацией.
## Реализация
### Access token и Refresh token 
Содержаться в Cookies и называются at и rt соответственно. 
Cодержат: 
``` json
{
	"userId": 6789,
	"role": "ADMIN", // STUDENT | RESIDENT | COMMANDANT | ADMIN | MODERATOR | DEVELOPER
}
```

| Role       | Доступы                                                                    |
| ---------- | -------------------------------------------------------------------------- |
| STUDENT    | Афиша <br><br>Услуги <br><br>Каталог <br><br>Личный чат <br><br>Расписание |
| RESIDENT   | Чат общежития                                                              |
| COMMANDANT | Верификация пользователей<br><br>Система уведомлений                       |
| ADMIN      | Дозволено все в пределах ВУЗа                                              |
| MODERATOR  | Полный доступ ко всему с целью модерирования нежелательного контента       |
| DEVELOPER  | Доступ к бд                                                                |

### Уведомления
Надо выбрать либо Server send events

По идее можно сделать так, чтобы пользователи искались по ФИО. Комендант копирует ФИО пользователей из своих таблиц и отсылает уведомление каждому с таким ФИО. Надо обрабатывать коллизию ФИО.
